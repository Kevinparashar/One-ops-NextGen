# `ops/pmg-evidence/` — PMG validation evidence

Every PMG-validatable claim in the Day-1 cut writes a log file here. The PMG demo runbook (`docs/runbooks/pmg-demo-runbook.md`) links every claim to a file under this directory. `make pmg-verify` runs `verify-all.sh`, which iterates the phases and writes the master report to `REPORT.md`.

## Convention

```
ops/pmg-evidence/
├── README.md                    ← this file
├── verify-all.sh                ← master verifier, run via `make pmg-verify`
├── REPORT.md                    ← generated by verify-all.sh on each run
├── phase-N-name.log             ← per-phase deliverable evidence log
├── traces/                      ← Tempo trace IDs + JSON dumps
├── dashboards/                  ← Grafana dashboard JSON exports
├── screenshots/                 ← PNG screenshots (Grafana, Tempo, terminal)
└── manifests/                   ← AgentManifest export bundles + round-trip records
```

## Per-phase artefacts produced

| Phase | Log file | Folder artefacts |
|---|---|---|
| 1 Scaffolding | `phase-1-scaffolding.log` | — |
| 2 UC-5 Triage | `phase-2-uc05-routing.log` | `traces/uc05-*.json` (5 trace IDs from real T001 incidents) |
| 3 Lifecycle state machine | `phase-3-lifecycle.log` | `traces/lifecycle-active.json`, `traces/lifecycle-refused.json` |
| 4 UC-8 Fulfillment | `phase-4-uc08-fulfillment.log` | `traces/uc08-onboarding.json` (full wave→interrupt→resume tree) |

| 6 Local CI gate | `phase-6-ci-gate-green.log`, `phase-6-ci-gate-blocks.log` | — |
| 7 Demo runbook + decision package | (the runbook + decision package docs themselves) | — |

## Rules this directory honours

- **§2.7 no silent failures** — every `verify-all.sh` step exits non-zero on failure; the master report shows red rows visibly.
- **§2.10 no file bloat** — only PMG-relevant artefacts land here; intermediate scratch goes to `/tmp/`.
- **§2.6 observability** — log files are operator-grade plain text + structured trace JSON, not PNG-only narratives. PMG can grep them.

## Reading order for a reviewer

1. Open `REPORT.md` — one row per phase, status green/red, link to evidence.
2. For any red row, open the matching `phase-N-*.log`.
3. For deep verification, open the trace JSON in `traces/` and load into Tempo / Jaeger.
